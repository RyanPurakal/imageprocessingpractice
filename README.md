# Image restoration lab

Small demo that ties together **PyTorch**, **Hugging Face Transformers**, synthetic **mechanics-flavored** 2D fields (smooth scalar “slices” you might get from CFD/postprocessing), **optional super-resolution**, a minimal **R** plotting script, and a tiny **open-weights language model** example (DistilGPT-2).

It does **not** run FEM, SPH, or a CFD solver; it is meant as a bridge project between your simulation/visualization world and modern ML tooling. You can swap the synthetic tensors for PNG exports from a C++ or Python postprocessor.

## Architecture

```
┌─────────────────────────────────────────────────────────────────┐
│                          Entry points                           │
│   train.py  ──────────────────────────────────  infer.py        │
│       │                                              │           │
└───────┼──────────────────────────────────────────────┼──────────┘
        │                                              │
        ▼                                              ▼
┌───────────────────┐                    ┌─────────────────────────┐
│ src/train_denoise │                    │  src/infer_denoise      │
│  - generates data │                    │  - loads checkpoint     │
│  - trains U-Net   │                    │  - runs model           │
│  - writes CSV     │                    │  - saves figure         │
└────────┬──────────┘                    └──────────┬──────────────┘
         │                                          │
         ▼                                          ▼
┌────────────────────────────────────────────────────────────────┐
│                        src/ package                            │
│                                                                │
│  synthetic_fields.py ──► add_gaussian_noise                    │
│  field_io.py         ──► load PNG/PGM from disk                │
│        │                        │                              │
│        └──────────┬─────────────┘                              │
│                   ▼                                            │
│             unet.py  (UNetDenoise1ch)                          │
│                   │                                            │
│                   ▼                                            │
│             metrics.py  (batch_psnr)                           │
└────────────────────────────────────────────────────────────────┘
         │                                          │
         ▼                                          ▼
  checkpoints/denoiser.pt               outputs/denoise_*.png
                                        outputs/metrics.csv
```

**Data flow (training):** `synthetic_fields` generates clean batches → noise is added → `UNetDenoise1ch` predicts the clean image → MSE loss drives Adam → checkpoint saved.

**Data flow (inference):** Checkpoint loaded → clean images sourced from either `synthetic_fields` or `field_io` (PNG/PGM files) → noise added → model restores → PSNR computed and comparison figure saved.

**Optional paths:**
- `hf_swin2sr.py` — standalone 2× super-resolution via Hugging Face Swin2SR; downloads weights on first run.
- `optional_llm_blurb.py` — text generation with DistilGPT-2; unrelated to the image pipeline.
- `r/plot_metrics.R` — reads `outputs/metrics.csv` and writes `outputs/training_curves.png` using base R.
- `examples/export_pgm_stub.cpp` — C++17 reference showing how a simulation postprocessor can write PGM files that `infer.py --png` can then consume.

## Directory map

| Path | Responsibility |
|------|---------------|
| `src/` | Core ML package — model, data, training loop, inference, I/O, metrics |
| `scripts/` | Shell automation (`demo.sh` runs the full end-to-end pipeline) |
| `tools/` | Dev utilities (write synthetic PNGs to `sample_data/`) |
| `examples/` | C++17 PGM export stub showing the simulation → Python handoff |
| `r/` | Optional R script for plotting training curves |
| `sample_data/` | Input images for the PNG inference path (gitignored except `.gitkeep`) |
| `outputs/` | Generated figures and metrics CSV (selected files committed for GitHub rendering) |
| `checkpoints/` | Saved model weights (gitignored) |

## Stack

| Area | Technologies |
|------|----------------|
| **Core** | Python 3.10+ |
| **ML / DL** | PyTorch, torchvision |
| **Pretrained models** | Hugging Face Transformers, Accelerate (Swin2SR super-resolution) |
| **Numerics & images** | NumPy, Pillow |
| **Training / UX** | tqdm |
| **Visualization** | Matplotlib (figures); optional R + base graphics (`r/plot_metrics.R`) |
| **Optional demos** | DistilGPT-2 via Transformers pipelines; C++17 PGM export stub (`examples/`) |
| **Automation** | Bash (`scripts/demo.sh`), Make (`Makefile`) |

**Not included:** FEM, SPH, or CFD solvers — this repo is a small bridge from scalar-field-style images to denoising / SR tooling.
